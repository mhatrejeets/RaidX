# 🏆 RaidX

## Real-Time Kabaddi Live Scoring & Tournament Platform

RaidX is a **real-time Kabaddi match scoring and streaming platform** designed for tournament organizers, scorers, teams, and spectators.

The system enables:

* ⚡ **Live raid-by-raid scoring**
* 📡 **Real-time score broadcasting**
* 🏟 **Tournament & championship management**
* 👥 **Team and player management**
* 🔐 **JWT based authentication**
* 🛂 **Resource scoped RBAC authorization**
* ✉️ **Invitation driven participation**
* 🚀 **Low latency match event streaming**

RaidX is designed as a **real-time sports event streaming backend** where scoring events generated by the scorer interface propagate instantly to all connected viewers.

The backend is implemented in **Go using Fiber**, uses **MongoDB for persistence**, **Redis for runtime state**, and **WebSockets for real-time streaming**.

# 🔐 Authentication

RaidX uses **JWT based stateless authentication**.

After login, users receive a **signed JWT token** which must be attached to all protected requests.

Supported token locations:

* Authorization header
* Query parameter
* HTTP cookie

---

## Authentication Flow

```
User
 │
 │  Login
 ▼
Client Application
 │
 │ POST /login
 ▼
API Server
 │
 │ Validate Credentials
 ▼
MongoDB
 │
 │ Generate JWT
 ▼
JWT Service
 │
 │ Return Token
 ▼
Client

Client ── Request + Token ──► API
API ── Verify JWT ──► Allow / Reject
```

Authentication middleware validates tokens before protected routes execute.

---

# 🛂 Authorization Model

RaidX implements **resource scoped RBAC**.

Authorization decisions depend on:

* 👤 Global user role
* 🏟 Resource ownership
* 👥 Team membership
* 🏆 Event participation
* ✉️ Invitation status

Instead of static role → permission tables, RaidX determines access dynamically using **relationships between users and resources**.

---

## RBAC Relationship Model

```
Users
 │
 ├── Own ───────────────► Teams
 │                         │
 │                         └── Players
 │
 ├── Organize ──────────► Events
 │                         │
 │                         └── Participating Teams
 │
 └── Receive ───────────► Invitations
                           │
                           ├── Team Invitations
                           └── Event Invitations
```

This approach makes RBAC **flexible and resource aware**, especially for tournament structures.

# ⚡ Live Scoring System

RaidX provides a **scorer interface** used during live matches.

Scorers record raid outcomes which are transmitted to the backend.

The backend processes the event and broadcasts updates to viewers.

---

## Live Scoring Flow

```
Scorer Interface
        │
        │ Raid Event
        ▼
API Server
        │
        ├── Update Runtime State (Redis)
        │
        ├── Persist Match Data (MongoDB)
        │
        ▼
WebSocket Broadcast
        │
        ▼
Connected Viewers
```

This architecture ensures **instant score propagation**.

---

# 📡 WebSocket Streaming

RaidX streams match updates to connected viewers through WebSockets.

Viewers subscribe to match specific channels.

Example endpoint pattern:

```
/ws/match/{matchId}
```

Events broadcast include:

* score updates
* raid results
* player status changes
* match completion

This allows large numbers of spectators to receive updates simultaneously.

---

# ⚡ Redis Runtime Layer

Redis is used for fast changing runtime data.

Typical usage:

* live match state
* scoring session state
* WebSocket broadcast synchronization

Redis reduces load on MongoDB during active matches.

# 🧠 Design Principles

RaidX architecture follows several design principles:

### ⚡ Event Driven Updates

Match state changes are propagated through events rather than direct state mutations.

### 🔐 Stateless Authentication

JWT based authentication allows horizontal scaling of API servers.

### 🛂 Resource Scoped Authorization

Permissions derive from relationships between users and resources.

### 🧩 Modular Backend Architecture

Handlers, middleware, models, and infrastructure layers are clearly separated.

### 📡 Real-Time Streaming

WebSocket architecture enables low latency score updates.
